Security Hardening (สำหรับ GitHub Pages)

  1. ลดการพึ่งพาภายนอก: ใช้ asset แบบ local ทั้งหมด, inline CSS/JS เฉพาะที่จำเป็น
  2. CSP (meta): กำหนด default-src 'self', object-src 'none', frame-ancestors 'none', upgrade-insecure-requests
  3. Referrer-Policy: no-referrer
  4. Permissions-Policy: ปิดการเข้าถึงกล้อง/ไมค์/ที่อยู่ (ตั้งได้ผ่าน HTTP header ที่ proxy เช่น Cloudflare)
  5. ไฟล์สำคัญ: เพิ่ม /.well-known/security.txt และ robots.txt (หากใช้ custom domain + proxy)
  6. Automated checks: ใช้ GitHub Advanced Security (secret scanning/Dependabot) และเปิด Branch protection
ขั้นตอนตั้งค่า Repo และ Pages (แนะนำ)
  1. สร้าง repo ชื่อ nhoom-apichai.github.io (Public)
  2. วางไฟล์นี้เป็น index.html ที่ root แล้ว commit & push
  3. เปิด GitHub Pages: ไปที่ Settings → Pages เลือก Deploy from main
  4. (ตัวเลือก) ใช้ Cloudflare Proxy เพื่อเพิ่ม Security Headers ระดับ HTTP
  5. เปิด Settings → Code security and analysis: เปิด Dependabot, Secret scanning
  6. เปิด Settings → Branches: เพิ่ม Branch protection (Require PR, status checks)

ตัวอย่างคำสั่ง Git:

git init

git remote add origin https://github.com/nhoom-apichai/nhoom-apichai.github.io.git
